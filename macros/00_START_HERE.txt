THE CORE CONCEPT IN 2 MINUTES
================================================================================

Problem: AI systems don't understand SAS macros

Solution: Create a JSON registry that describes each macro

Result: AI can automatically discover, select, order, and call macros

The Key File: macro_registry.json
  - Contains all macro metadata
  - Machine-readable format (JSON)
  - Tells AI orchestrator:
    * What macros exist
    * What each macro does
    * What parameters each needs
    * How macros depend on each other
    * When to use each macro

Example AI Orchestration Flow:
  1. AI reads macro_registry.json
  2. AI analyzes input dataset structure
  3. AI matches variables to macro purposes (from registry)
  4. AI checks dependencies (from registry)
  5. AI generates SAS code
  6. AI executes the code
  7. AI validates output

This enables autonomous clinical data transformation with human oversight!

QUICK EXAMPLES

EXAMPLE 1: Convert a date variable to ISO 8601
──────────────────────────────────────────────

%include "/path/to/iso_date.sas";

%iso_date(
  inds=raw.dm,
  outds=work.dm_iso,
  invar=BRTHDT,
  outvar=BRTHDTC
);

Result: BRTHDT (various formats) → BRTHDTC (YYYY-MM-DD format)


EXAMPLE 2: Calculate age from dates
────────────────────────────────────

%include "/path/to/derive_age.sas";

%derive_age(
  inds=work.dm_iso,
  outds=work.dm_aged,
  brthdt=BRTHDTC,
  refdt=RFSTDTC,
  agevar=AGE
);

Result: New AGE variable with age in years
         New AGEU variable with value='YEARS'


EXAMPLE 3: Map raw SEX values to CDISC CT
──────────────────────────────────────────

%include "/path/to/assign_ct.sas";

%assign_ct(
  inds=work.dm_aged,
  outds=work.dm_final,
  invar=RAW_SEX,
  outvar=SEX,
  codelist=C66731,
  ctpath=/path/to/ct_lookup.csv
);

Result: RAW_SEX (M, MALE, 1, F, FEMALE, etc.)
        → SEX (M, M, M, F, F, etc.) [standardized]

================================================================================
MACROS AT A GLANCE
================================================================================

%iso_date
  Purpose: Convert various date formats to ISO 8601 (YYYY-MM-DD)
  Auto-detects: DD-MON-YYYY, MM/DD/YYYY, DD/MM/YYYY, DDMONYYYY
  Output: Character variable in YYYY-MM-DD format
  Key Parameters: inds, outds, invar, outvar, infmt(optional)
  Dependencies: None
  When to use: Before age derivation, before any SDTM date work

%derive_age
  Purpose: Calculate age in years from birth and reference dates
  Algorithm: YRDIF function (accurate age in completed years)
  Output: Numeric AGE variable + optional character AGEU variable
  Key Parameters: inds, outds, brthdt, refdt, agevar
  Dependencies: Requires ISO-formatted dates from %iso_date
  When to use: Deriving AGE in Demographics domain

%assign_ct
  Purpose: Map raw values to CDISC Controlled Terminology
  Supports: SEX (C66731), RACE (C74457), ETHNICITY (C66790), COUNTRY (C71113)
  Output: Character variable with standardized CT values
  Key Parameters: inds, outds, invar, outvar, codelist, ctpath
  Dependencies: Requires ct_lookup.csv file
  When to use: Standardizing categorical variables to CT
